Responsive Redesign: Reimagining the Yale School of Arts Website

Overview

The goal of this project is to identify and analyze flaws in an existing interface, create low-fidelity and high-fidelity prototypes for various screen sizes,and build a responsive website based on the prototypes.

Identifying Usability Problems


For my project, I am redesigning the Yale School of Arts website.The website has a generally overwhelming layout, and for a website that is frequented by most, this presents a confusing user journey to a wide variety of users, including those that frequent the site.
At first glance , a lot of information is represented to the user with a lot of background distraction that makes it hard to decipher what the website is all about.
Visit Yale School of Arts to access the school of arts' original website.

Finding Problems

1.The webpage presents a navigation problem since the menu is not clearly identifiable to a new user. This presents a confusing user journey when attempting to navigate to different pages. This could potentially be solved by having a menu icon that is more memorable to a user.
2.The inconsistent layout and the background graphics are generally overwhelming and distracting to the user. The background graphics present readability challenges to the user. For new users, the inconsistent layout makes it difficult to know where to get specific information when they want.The inconsistency in layout makes it difficult to identify what segments are news and which ones are not.
3. The overall visual design of the page is lacking in appeal and aesthetics as information is not organized neatly.
4.Information presented in the community bulletin to provide an overview of the page can be overwhelming to a user since it is not simplified

Accessibility

I tested the page on WebAIM WAVE to detect possible accessibility flaws.WebAIM WAVE detected long alternative text on the community bulletin board, in addition to long description texts for the images. The long texts definitely overwhelm a user once they land on the page. The page also had multiple redundant links that only made the page cluttered. This can be offset by having all the relevant links in one menu.

Visual Design Style Guide

To guide my high -fidelity prototyping, I began by preparing a visual design style guide. This guide contains information about the color palette, the typography ,and the reusable components' different states on the website.


High-fidelity prototyping

For the hi-fi prototyping, I decided to organize the elements in flexboxes.The header and the first paragraph are in different flexboxes and so is the body. The body is one big flexbox containing the different sections of news, represented by images in my prototype.For my redesign, there is no need to adjust font sizes in between the different screen sizes as the initial font size is very adaptable on all three options of screen sizes. Upon clicking the Read More News button, more sections of news unravel .The purpose of the button is to initially prevent endless scrolling and therefore prevent a user from getting overwhelmed.
